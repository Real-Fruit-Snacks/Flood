01 Premise

Overwhelming surge of concurrent requests to discover every hidden endpoint.

Flood is an async Rust web fuzzer that sends 100 concurrent requests to systematically discover directories, virtual hosts, and parameters. FUZZ keyword placement with recursive scanning, auto-throttling, and advanced filtering.

Built for comprehensive discovery through intelligent concurrency management and real-time interactive controls.

03 Quickstart

Clone, build, fuzz, discover.

Flood handles directory enumeration, VHost discovery, and parameter fuzzing with intelligent concurrency.

# Prerequisites: Rust toolchain, wordlists
$ git clone https://github.com/Real-Fruit-Snacks/Flood.git
$ cd Flood && make release

# Directory enumeration with FUZZ keyword
$ ./target/release/flood -u https://target.com/FUZZ -w common.txt

# Virtual host discovery
$ ./target/release/flood -u https://target.com -H "Host: FUZZ.target.com" -w subs.txt

# Recursive scanning with extensions
$ ./target/release/flood -u https://target.com/FUZZ -w dirs.txt -r -e php,bak --depth 3

Fuzzing modes and filtering options.

PARAMETER FUZZING

-d "user=admin&pass=FUZZ"POST data parameter fuzzing
-u https://target.com/search?q=FUZZGET parameter fuzzing

FILTERING & MATCHING

--match-code 200,301,403Match specific status codes
--filter-code 404,429Filter out status codes
--match-size 1337-1400Match response size range
--match-regex "admin|dashboard"Match regex in response
--filter-regex "not found|404"Filter regex patterns

RECURSIVE & EXTENSIONS

-r --depth 3Recursive scanning with max depth
-e php,asp,jsp,txt,bakFile extensions to append
--discover-backupAuto-discover backup files

PERFORMANCE & OUTPUT

05 Architecture

Three-layer async engine.

CLI config parser feeds async fuzzing engine which fans out requests across tokio tasks, while filter pipeline processes responses with match/filter criteria and auto-throttling.

src/
main.rs          // CLI dispatch and config parsing
fuzzer/          // async fuzzing engine + tokio
http/            // HTTP request/response handling
filter/          // match/filter pipeline
output/          // JSON · CSV · JSONL formatters
throttle/        // auto-throttle + rate limiting
recursive/       // directory discovery + re-queue
ui/              // Catppuccin Mocha terminal UI
06 Authorization

Authorized web testing only.

Flood is designed for legitimate web application security testing with proper authorization. The tool generates significant HTTP traffic and server logs — ensure explicit permission.

Security vulnerabilities should be reported through private security advisories with 90-day responsible disclosure.

Flood does not exploit endpoints, inject payloads, or evade WAF/IDS systems.
