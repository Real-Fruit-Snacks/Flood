FEATURES

Fuzz directories, vhosts, and parameters

FUZZ keyword placement anywhere in the request. Recursive discovery, advanced filtering, proxy replay, auto-throttle.

Directory Enumeration

FUZZ keyword in any URL position. Automatic extension appending, recursive scanning into discovered directories, configurable depth limit.

$ flood -u https://target.com/FUZZ -w common.txt -r
200 /admin/
recursing into /admin/ — Ctrl+C to disconnect
100
Default Threads

VHost Discovery

FUZZ in Host header for subdomain enumeration. Filter by response size to strip default pages. Wildcard detection.

Parameter Fuzzing

FUZZ in POST data, query strings, cookies, or custom headers. Clusterbomb mode with FUZZ/FUZ2Z for multiple wordlists.

Relay & SOCKS5

Port forwarding, multi-hop chain relay, and proxychains-compatible SOCKS5 proxy server.

4
Output Formats

Auto-Throttle

Automatic backoff on 429 responses. Configurable rate limiting. Interactive thread adjustment during scans. Pause, resume, and save state for long-running fuzzes.

$ flood -u https://target.com/FUZZ -w big-list.txt
429 detected — throttling to 20 req/s
p: pause | q: save+quit | +/-: threads — padding 256B, jitter 50-150ms

Proxy Replay

Route all traffic through Burp Suite or replay only matched results to a proxy for manual inspection.

ARCHITECTURE

Three-layer design

CLI parses flags into FuzzConfig, async engine fans out requests across tokio tasks, filter pipeline processes responses.

1

CLI Layer

Argument parsing into FuzzConfig. FUZZ/FUZ2Z keyword detection, wordlist loading, filter chain assembly. Supports resume from saved state.

2

Async Engine

tokio runtime fans out HTTP requests. Semaphore-based concurrency control. Auto-throttle monitors 429s and adjusts rate transparently.

3

Filter Pipeline

Response processing chain: status code, size, word count, line count, regex matching. Matched results forwarded to output and optional replay proxy.

4

Output + UI

JSON, JSONL, CSV, text formatters. Catppuccin Mocha terminal with real-time progress, interactive controls, and state serialization.

GETTING STARTED

Build and fuzz

Single Rust binary. Build and start fuzzing in under a minute.

build
$ git clone https://github.com/Real-Fruit-Snacks/Flood.git
$ cd Flood && make release

# Static binary for target deployment
# Cross-compile

$ make release-linux
$ make release-windows
usage
# Directory enumeration $ flood -u https://target.com/FUZZ -w common.txt -r # VHost discovery $ flood -u https://target.com -H "Host: FUZZ.target.com" -w subs.txt # Parameter fuzzing $ flood -u https://target.com/login -d "pass=FUZZ" -w passwords.txt
DETECTION AWARENESS

Know the boundaries

Hidden From

  • Proxy integration — route through Burp for analysis
  • Rate limiting — configurable req/s cap
  • Auto-throttle — automatic 429 backoff
  • Recursive depth limit — prevents infinite loops
  • State save — resume interrupted scans

Visible To

  • WAF/IPS — request pattern detection
  • Rate-based detection — high request volume
  • Access logs — all requests logged server-side
  • Response codes — mass 404s visible in logs
  • IP blocking — source IP visible to target
